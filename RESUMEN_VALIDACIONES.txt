3️⃣  storeapp/exceptions/ValidationException.java
    └─ Excepción genérica de validación
    └─ Lanzada cuando: Datos inválidos, campos vacíos, formatos incorrectos
    └─ Hereda de: RuntimeException

4️⃣  storeapp/utils/EntityValidator.java
    └─ Clase UTILITARIA centralizada de validaciones
    └─ Contiene: 15 métodos estáticos de validación
    └─ Valida: Email, nombres, precios, stocks, IDs, contraseñas, etc.
    └─ Métodos principales:
       • isValidEmail()          → Valida formato email
       • isValidPrice()          → Valida precio > 0
       • isValidStock()          → Valida stock >= 0
       • isValidPassword()       → Valida contraseña >= 6 caracteres
       • isValidName()           → Valida nombre >= 2 caracteres
       • validateNotEmpty()      → Lanza excepción si está vacío
       • validatePositivePrice() → Lanza excepción si precio <= 0

5️⃣  storeapp/utils/DuplicateChecker.java
    └─ Clase UTILITARIA para detección de duplicados
    └─ Contiene: 8 métodos estáticos de verificación
    └─ Métodos principales:
       • checkCustomerDuplicateById()      → Verifica cliente ID único
       • checkCustomerDuplicateByEmail()   → Verifica email cliente único
       • checkCategoryDuplicateById()      → Verifica categoría ID único
       • checkProductDuplicateById()       → Verifica producto ID único
       • customerExists()                  → ¿Existe cliente?
       • categoryExists()                  → ¿Existe categoría?
       • productExists()                   → ¿Existe producto?

🏗️ ARQUITECTURA - CÓMO FUNCIONA:

USER (MenuApp)
    ↓
    ↓ Ingresa datos
    ↓
SERVICE (CustumerServiceImpl, CategoryServicesImpl, ProductServiceImpl)
    ├─ Llamada a EntityValidator (validar campos)
    │   └─ Lanza ValidationException si hay errores
    │
    ├─ Llamada a DuplicateChecker (verificar duplicados)
    │   ├─ Lanza DuplicateIdException si ID existe
    │   └─ Lanza DuplicateEmailException si email existe
    │
    └─ Si todo OK → REPOSITORY (guardar datos)
                        └─ CustomerRepository/CategoryRepository/ProductRepository
                            └─ ArrayList (en memoria)

CATCH BLOCKS:
    ├─ catch(DuplicateIdException)    → Mensaje específico ID duplicado
    ├─ catch(DuplicateEmailException) → Mensaje específico email duplicado
    ├─ catch(ValidationException)     → Mensaje específico de validación
    └─ catch(Exception)               → Error genérico inesperado
